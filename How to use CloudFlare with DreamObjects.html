This tutorial covers how to configure CloudFlare and a DreamObjects bucket to work together so that you can accelerate/protect content in the bucket.

This solution requires that the root domain be using CloudFlare's nameservers.

Creating the bucket

The name of the bucket must be the same name of the DNS entry for CloudFlare. For example, if you want to use cdn.example.com to access accelerated/protected DreamObjects content, name the bucket cdn.example.com:

[user@localhost]$ s3cmd mb s3://cdn.example.com
Bucket 's3://cdn.example.com/' created

Creating the DNS entry

  • Type: CNAME
  • Name: must exactly match the bucket you created
  • Domain name: BUCKET.objects-us-west-1.dream.io
For more information about adding CNAME records in CloudFlare, see their documentation here.

As soon as DNS information propagates, your content is reachable using the hostname you specified.
